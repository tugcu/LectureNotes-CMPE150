Function Overloading in Python

!!!

Requires decorator usage

https://www.codementor.io/@arpitbhayani/overload-functions-in-python-13e32ahzqt

In the following example, there are three different functions with the same name, but different signatures, which makes them different functions. However, it is not allowed to have more than one functions with the same signature.

import turtle

def draw_a_square_for_me():
    draw_a_square_for_me(100)
	
def draw_a_square_for_me(square_size):
    turtle.forward(square_size)
    turtle.right(90)
    turtle.forward(square_size)
    turtle.right(90)
    turtle.forward(square_size)
    turtle.right(90)
    turtle.forward(square_size)
    turtle.right(90)

def draw_a_square_for_me(length_of_a_side, color):
    turtle.pencolor(color)
    draw_a_square_for_me(length_of_a_side)
  • The first definition of draw_a_square_for_me does not receive any arguments, and it makes a call to the second definition of the function with a default value of 100 as the square_size argument.
  • The second definition of the function, with a different signature does the drawing as we practiced earlier.
  • The last definition has an additional argument that can be used for changing the pen color of the turtle marker, and pass the received length_of_a_side argument as is to the second definition of the function.
  • This concept is known as function overloading by using different signatures.
  • Although the argument names are different, the following use is not allowed, since the signatures are exactly the same:
def draw_a_square_for_me(square_size):
    turtle.forward(square_size)
    turtle.right(90)
    turtle.forward(square_size)
    turtle.right(90)
    turtle.forward(square_size)
    turtle.right(90)
    turtle.forward(square_size)
    turtle.right(90)

def draw_a_square_for_me(length_of_a_side):
    turtle.pencolor("Red")
    draw_a_square_for_me(length_of_a_side)
